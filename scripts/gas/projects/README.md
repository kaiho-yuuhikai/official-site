# 事業ID自動採番 Apps Script (フォーム① onSubmit)

事業計画申請フォーム（フォーム①）の応答スプレッドシートに紐付けて、
送信トリガーで `①-001`, `①-002`, … の連番IDを採番し、申請者へ自動返信メールを送る。

## 構成ファイル

- `Projects.js` — 採番ロジック + onFormSubmit ハンドラ
- `appsscript.json` — マニフェスト（タイムゾーン JST、V8 ランタイム）

## シート列定義

| 列 | 内容 | 入力 |
|---|---|---|
| 1 | タイムスタンプ | フォーム自動 |
| 2 | メールアドレス | フォーム |
| 3 | **事業ID** | **GAS が採番** |
| 4 | 事業名 | フォーム |
| ... | フォーム項目 | フォーム |

事業ID列は事前に空列として作成しておくこと。

## セットアップ手順

```bash
# 0. clasp 認証
clasp logout && clasp login

# 1. フォーム①応答シートに紐付く Apps Script プロジェクトを clone
cd scripts/gas/projects
clasp clone <スクリプトID>

# 2. push
clasp push
```

### トリガー設定

GAS エディタで「トリガー」→「トリガーを追加」:
- 関数: `onFormSubmit`
- イベントのソース: スプレッドシートから
- イベントの種類: フォーム送信時

これで、フォーム送信のたびに採番＋メール送信が走る。

## テスト

ローカル（Node + Vitest）で採番ロジックを検証:

```bash
npm test
```

`tests/projects-id.test.ts` が `Projects.js` を `vm` で評価し、
`SpreadsheetApp` / `MailApp` をモックした上でロジック検証する。

## 採番規則

- `①-001` から始まる単調増加
- 既存IDの最大値 + 1
- 歯抜けは埋めない（削除された行があっても次番に進む）
- 規則外の値（`②-...` や空、`hello` 等）は無視

## メール内容

```
件名: [開邦雄飛会] 事業ID ①-XXX を発行しました
本文:
事業計画申請を受け付けました。

事業ID: ①-XXX
事業名: <フォームで入力された事業名>

事業実施後は、報告フォームで上記の事業IDを入力してください。

開邦雄飛会
```

メール文面の修正は `composeNotificationEmail()` を編集する。
