# サイト公開（GitHub Pages へデプロイ）

ローカルの変更を GitHub Pages に反映します。

## 手順

### Step 1: 変更内容の確認

```bash
git status
git diff
```

変更内容の概要をユーザーに日本語で分かりやすく説明してください。
技術的な詳細ではなく、「何がどう変わるか」をビジネスの観点で伝えてください。

### Step 2: ユーザーに最終確認

以下を確認してください:
- ローカルプレビューで変更内容を確認済みですか?
- この内容で公開サイトに反映して問題ありませんか?

ユーザーが「OK」「はい」「問題ない」等の肯定的な回答をした場合のみ、次のステップに進んでください。

### Step 3: コミット & プッシュ

変更内容に基づいて適切なコミットメッセージを日本語で作成し、コミット・プッシュしてください。

```bash
git add -A
git commit -m "適切なコミットメッセージ"
git push origin main
```

コミットメッセージの規約:
- 新機能追加: `feat: 〇〇を追加`
- 既存機能の更新: `update: 〇〇を更新`
- テキスト・コンテンツ修正: `content: 〇〇を修正`
- デザイン変更: `style: 〇〇のデザインを変更`
- バグ修正: `fix: 〇〇を修正`

### Step 4: デプロイ確認

プッシュ後、GitHub Actions の実行状況を確認してください。

```bash
gh run list --limit 1
```

ユーザーに以下を案内:
- GitHub Actions でビルド・デプロイが自動実行されます（約2-3分）
- デプロイ完了後、公開サイトに反映されます

## 注意事項

- `main` ブランチへの push で自動デプロイが実行されます
- デプロイに失敗した場合は GitHub Actions のログを確認してください
